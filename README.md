🎮 ModRandomizer v4.1 - 手动分组优先版

基于WWMI的mod使用原理为每个角色随机选择一个mod,支持排除特定mod,支持角色多皮肤列表
======================================

【核心设计理念】
✅ 手动为主：用户完全掌控分组数量、关键词规则、Mod 归属
✅ 自动为辅：关键词自动分组仅作为辅助建议，可一键执行
✅ 手动优先：用户手动调整结果永久覆盖自动分组，永不丢失
✅ 灵活归类：支持拖拽式重归类，未分组单独管理

【分组操作流程】
1️⃣ 创建分组
   • 双击角色 → 打开皮肤分组配置
   • 点击"➕ 新增分组" → 输入名称（如"夏日皮肤"）
   • 重复创建所有需要的分组

2️⃣ 设置关键词规则（可选）
   • 选中分组 → 在"关键词规则"输入框填写
   • 支持多种分隔符：逗号、分号、空格
   • 支持通配符：* 匹配任意字符，? 匹配单个字符
   • 示例：
       - "夏日,summer,泳装" → 匹配包含任一关键词的 Mod
       - "winter*" → 匹配 winter、winter2023 等
       - "?万圣?" → 匹配 a万圣b、1万圣2 等

3️⃣ 执行自动分组（可选）
   • 点击"⚡ 执行自动分组" → 工具按关键词自动归类
   • 未匹配的 Mod 自动归入"❓ 未分组"

4️⃣ 手动调整（核心步骤）
   • 在"❓ 未分组"区域，点击 Mod 右侧的分组按钮快速分配
   • 或拖拽 Mod 到目标分组（界面支持直观操作）
   • 手动调整结果永久保存，不受后续自动分组影响
   • 点击"↺"可将 Mod 重置回"未分组"

5️⃣ 设置激活皮肤
   • 在分组树中点击目标分组 → 该分组成为"激活皮肤"
   • 随机化操作仅影响激活皮肤相关 Mod

6️⃣ 共享 Mod 配置（高级）
   • 点击"⚙️ 共享Mod配置" → 勾选需要跨皮肤共享的 Mod
   • 共享 Mod 会同时影响多个皮肤的随机化

【关键特性】
🛡️ 手动调整 > 自动分组：用户操作具有最高优先级
🛡️ 未分组隔离：未匹配关键词的 Mod 单独管理，避免误操作
🛡️ 激活皮肤隔离：仅操作当前皮肤相关 Mod，冻结皮肤完全不受影响
🛡️ 完全离线：无网络请求，100% 本地操作
🛡️ Python 3.14 兼容：无废弃 API，标准库实现

【典型场景】
场景1：角色"艾尔莎"有混乱命名的皮肤 Mod
  • 创建分组：夏日、冬日、万圣节
  • 设置关键词：夏日组 → "泳装,海滩,summer"
  • 执行自动分组 → 大部分 Mod 自动归类
  • 手动调整剩余 5 个未分组 Mod → 完成分组

场景2：共享基础外观
  • 创建分组：战斗服、休闲服
  • 配置共享：将"基础外观"勾选为同时属于两个分组
  • 激活"战斗服" → 随机化时"基础外观"可被启用
  • 切换激活"休闲服" → "基础外观"依然可用
